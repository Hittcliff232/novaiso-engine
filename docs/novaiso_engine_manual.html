1. Обзор

NovaIso-Engine состоит из двух основных частей: редактор сцен и рантайм. Редактор отвечает за создание уровней, настройку света, UI, триггеров, шейдеров, ресурсов и упаковку проекта. Рантайм загружает упакованный проект, показывает splash screen, главное меню, сцену, HUD и обрабатывает Python-скрипты.

Что делает каждая кнопка редактора Какие действия умеют триггеры Какие методы доступны в Python Как пользоваться камерами, анимациями и AI builder Как устроен HTML UI и что он поддерживает
Важно: текущий HTML UI в движке не является встроенным браузером. Это собственный лёгкий runtime, который понимает ограниченный набор HTML-тегов, CSS-свойств и JS-присваиваний.

7. Viewport

Viewport показывает текущую сцену. Здесь же происходит перетаскивание сущностей, ручное размещение контента и большинство быстрых операций над объектами. В оверлее сверху выводятся служебные подсказки, а также счётчик анимаций.

Что можно делать мышью

  • ЛКМ: выбрать объект, триггер, свет, аудиоисточник или другой элемент сцены.
  • Перетаскивание: перемещение выделенного элемента.
  • ПКМ: вызвать контекстное меню под курсором.
  • Средняя кнопка: панорама камеры viewport.
  • Колесо: масштаб viewport.
  • Alt + drag: растягивание размеров для сущностей и trigger zone.

Что показывает оверлей

  • подсказки по управлению камерой и объектами;
  • состояние preview/edit режима;
  • информацию по zoom и сетке;
  • счётчик анимаций: сколько объектов имеют анимацию и сколько анимируются прямо сейчас.

11. Trigger Editor

Trigger Editor открывает граф логики для trigger zone или trigger resource. Логика строится из condition node и action node. Это основной визуальный инструмент для безскриптовой событийной системы.

ЭлементЧто делает
Add Condition NodeДобавляет проверку, которая должна вернуть true/false.
Add Action NodeДобавляет действие, исполняемое после успешных условий.
Custom ScriptПозволяет указать Python-функцию вместо встроенной логики.
Advanced List ViewПоказывает структуру триггера списком вместо свободного графа.
Save Trigger To LevelЗаписывает trigger graph прямо в текущий .niso.
Save AllСохраняет все изменения графа и связанных данных.
Если у action есть аргумент delay внутри JSON, действие не сработает мгновенно, а будет поставлено в очередь выполнения.

13. Resource Inspector

Resource Inspector редактирует содержимое выбранного файла ресурса. Поведение зависит от типа ресурса.

Изображение

  • Create Sprite Resource создаёт .nsprite по текстуре.
  • Create Object Resource создаёт .nobject.
  • Place In Viewport ставит картинку или объект в сцену.

Аудио

  • Play воспроизводит один раз.
  • Play Loop запускает по кругу.
  • Loop переключает флаг цикличности.
  • Stop останавливает проигрывание.
  • Place Audio Source создаёт источник в сцене.

Для .nsprite, .nobject, .nanim и .ntrigger инспектор показывает редактируемые поля, кнопку сохранения и команду Rename File From Resource Name.

У object resource есть preset combo со значениями none, player, enemy, friend, vendor, storage. При выборе пресета редактор может подставить стандартный скрипт. В object resource также сохраняются material-like параметры вроде Reflection, Pseudo 3D и Pseudo 3D Height.

Пример 1. Игрок, движение и прыжок

def on_spawn(entity, scene):
    scene.log("Игрок появился")
    scene.play_music("assets/audio/theme.ogg")

def on_update(entity, scene, dt):
    speed = 150.0
    vx, vy = entity.velocity()

    if scene.action_down("MoveLeft"):
        vx = -speed
    elif scene.action_down("MoveRight"):
        vx = speed
    else:
        vx = 0.0

    if scene.action_down("Jump") and entity.grounded():
        vy = -260.0

    entity.set_velocity(vx, vy)

def on_trigger(entity, scene, trigger_name):
    scene.log("Игрок вошёл в триггер: " + trigger_name)

Пример 2. Мерцание объекта через пользовательское свойство

def on_spawn(entity, scene):
    entity.set_float("phase", 0.0)

def on_update(entity, scene, dt):
    phase = entity.get_float("phase")
    phase += dt * 2.0
    entity.set_float("phase", phase)

    glow = 0.5 + 0.5 * __import__("math").sin(phase)
    entity.set_tint(1.0, glow, glow, 1.0)

Пример 3. Custom condition для триггера

def require_side(scene, trigger_name, args_json):
    return scene.camera_mode() == "side"

Пример 4. Custom action для триггера

import json

def open_secret(scene, trigger_name, args_json):
    args = json.loads(args_json or "{}")
    entity_id = args.get("entity", "secret_door")
    target = scene.find_entity(entity_id)
    if target.valid():
        target.set_visible(True)
        target.set_active(True)
        scene.log("Открыта секретная дверь: " + entity_id)

Пример 5. Катсцена: включить камеру и проиграть анимацию

def on_trigger(entity, scene, trigger_name):
    if trigger_name != "intro_zone":
        return

    scene.activate_virtual_camera("intro_cam")
    scene.set_camera_zoom_smooth(0.82, 3.5)

30. Ограничения и текущие рамки

Этот раздел важен, потому что мануал должен описывать текущую реальность движка, а не только желаемое поведение.
  • HTML UI не является браузером и не поддерживает полноценный DOM, сторонние JS-библиотеки или сетевые API.
  • Набор trigger actions и conditions ограничен тем, что перечислено в разделе выше.
  • RT Lights в движке означает real-time shader lighting, а не физически корректную трассировку лучей и не полноценный path tracing.
  • Reflection даёт стилизованный мокрый/глянцевый блик от ламп, а не полное зеркальное отражение сцены.
  • Pseudo 3D строится как 2.5D-экструзия спрайта и зависит от активной virtual camera, а не от настоящей 3D-геометрии.
  • Shader Stack видит только пользовательские постэффекты; внутренние шейдеры движка скрыты и не должны добавляться в список эффектов.
Если дальше захочешь, следующий логичный этап для документации: добавить отдельные разделы по формату project.json, подробной схеме .niso, таблице локализационных ключей и справочник по каждому встроенному preset script.
